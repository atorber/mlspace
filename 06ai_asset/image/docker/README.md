# Docker 规范

本目录对应镜像模块中的 Docker 能力，定义机器学习容器镜像在平台中的基础约定。

## 建议基线

- 明确基础镜像
- 在适用时明确 CUDA 与 CUDNN 版本
- 固定 Python 版本
- 使用可确定性的依赖安装流程
- 明确区分构建期依赖与运行期依赖

## 平台边界

本目录关注训练与服务镜像的构建基线和可追溯字段。
Kubernetes 运行时访问模式归 `ai_compute_resource/`，开发环境准备归 `dev_instance/`。

## 必需记录

每一个训练或服务镜像都应暴露以下信息：

- 镜像标签
- 基础镜像
- CUDA 栈版本
- Python 版本
- 依赖锁定来源

这些字段都应在实验 manifest 中被引用。
