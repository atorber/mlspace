# 业务模板模块

本模块主要对应百度百舸 AI 计算平台中的 `WorkFlow` 和业务流程落地场景，用于承接面向具体业务场景的项目模板、实验模板和解决方案骨架。

## 该模块管什么

- 面向业务的项目骨架（让团队更快从“想法”落到“可运行工程”）
- 业务级别的初始化模板（如数据字段映射、训练配置项默认值、评测口径）
- 将 `ai_asset/dataset/`、`job/`、`serving/`、`governance/` 串成一条对业务友好的落地路径

## 建议的模板目录结构

```text
workflow/
  <business_line_or_task>/
    project_template/          # 从该模板生成的项目骨架说明与示例
    experiment_template/       # 面向业务实验的配置与入口建议
    serving_template/          # 面向业务上线的推理接入与验证清单
    README.md                  # 该业务模板的适用范围、前置条件与使用说明
```

## 与治理记录的关系

业务模板不应替代 `governance/manifest/`，而是把业务落地过程中“必须落记录”的点显式化：

- 每次关键实验都产出 manifest，并能明确关联数据版本、环境基座和产物位置
- 当业务需要上线时，把兼容性检查与产物交付清单按固定步骤呈现出来

## 示例（占位约定）

首轮可以先从“最少可用模板”开始：

- 给出该业务场景下数据集字段到训练输入的映射示例
- 给出训练配置中关键参数（如 seed、精度模式、批大小、学习率策略）的建议口径
- 给出一份“上线前检查清单”并指向 `serving/compatibility/` 与 `ai_asset/model/`
