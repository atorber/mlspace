# K8s Runtime

本目录对应资源管理模块中的 Kubernetes 运行时能力，说明机器学习工作负载中用于访问云存储的 sidecar 模式。

## 典型场景

- 将对象存储挂载到训练 Pod 中
- 提供凭证刷新或令牌交换能力
- 将存储访问逻辑与主训练容器隔离

## 边界规则

- 主容器负责训练逻辑
- sidecar 负责存储访问或同步行为
- 数据集版本选择必须保持显式，不能隐藏在 sidecar 默认配置中

## 平台边界

本目录关注运行时资源接入和存储访问模式。
镜像构建归 `ai_asset/image/`，训练任务编排归 `job/`，模型上线兼容检查归 `serving/`。

## 运维建议

1. 在 manifest 中记录挂载的数据版本与存储端点。
2. 将 sidecar 配置纳入版本控制。
3. 避免 sidecar 隐式写入已提升的数据集目录。
